进制转换是一些非常基础的计算机知识，这篇文章主要说说十进制的小数转换为二进制的方法。了解十进制的小数转换为二进制，有助于明白为什么编程语言里面总是强调浮点数运算不精确的问题。

大家都知道十进制整数转换为二进制非常简单，只需要采用"除2取余，逆序排列"的方法即可。但小数的转法呢，却需要换一种方式："乘2取整，顺序排列"。

具体做法是：用2乘十进制小数，可以得到积，将积的整数部分取出，再用2乘余下的小数 部分，又得到一个积，再将积的整数部分取出，如此进行，直到积中的小数部分为零，或者达到所要求的精度为止。 然后把取出的整数部分按顺序排列起来，先取的整数作为二进制小数的高位有效位，后取的整数作为低位有效位。实例如下：

        题目：把0.75转换为二进制
            0.75
           x   2
          -------
            1.50 .... 取整数 1 后，再将小数部分乘以2   
             .50                      | 顺
           x   2                      | 序
          -------                     | 排
            1.00 .... 取整数 1        V 列
             .00 .... 小数部分为0，就搞定了

        答：0.75的二进制为：0.11

有的小数乘以2永远得不到一个整数，一般达到自己要求的精度后就会停止，所以咱们常说浮点数的运算不精确，比如：

        题目：把3.1转换为二进制

        解：整数3转换为二进制答案是：11，所以我们关注小数部分：
            0.1
           x  2
          -------
            0.2 .... 取整数 0 后，再将小数部分乘以2

             .2                      
           x  2                      
          -------                     
            0.4 .... 取整数 0

             .4                      
           x  2                      
          -------                     
            0.8 .... 取整数 0

             .8                      
           x  2                      
          -------                     
            1.6 .... 取整数 1

             .6                      
           x  2                      
          -------                     
            1.2 .... 取整数 1

             .2 （从这里开始一直循环下去）
           x  2                      
          -------                     
            0.4 .... 取整数 0

             .4                      
           x  2                      
          -------                     
            0.8 .... 取整数 0


        答：整数3转换为二进制答案是：11，所以3.1的二进制为：11.00011001100110011001100110011...无限循环下去。
        这也是为什么0.1 + 0.2 == 0.3却返回false的原因吧
